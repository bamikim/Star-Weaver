스타위버 MVP 구현 지시서
개발 환경: Godot 4 / GDScript / 모바일 (Android, iOS)

Task 1. 프로젝트 기본 세팅
Godot 4 프로젝트를 생성한다. 프로젝트 이름은 StarWeaver로 설정한다. 렌더러는 Mobile로 설정한다. 화면 해상도는 1080×1920 (세로형)으로 고정하고, stretch mode는 canvas_items, aspect는 keep로 설정한다. 다음 Autoload 싱글톤을 생성한다. GameManager (게임 전체 상태 관리), DataManager (재화 및 저장 관리), BattleManager (전투 진행 관리). 씬 구조는 다음과 같이 구성한다.
res://
├── scenes/
│   ├── main/       # 메인 홈 화면
│   ├── battle/     # 전투 화면
│   └── ui/         # 공통 UI 컴포넌트
├── scripts/
│   ├── autoload/
│   ├── player/
│   ├── enemy/
│   └── ui/
└── assets/
    ├── sprites/
    └── audio/

Task 2. DataManager 구현
DataManager.gd를 Autoload로 구현한다. 관리할 데이터는 다음과 같다.
재화 3종: dust (기억의 먼지), sacred_water (성수), seal (별의 인장). 연못 레벨: pond_level (1~20). 장비 슬롯 4종: weapon, armor, accessory, bag. 현재 스테이지: current_stage, current_chapter.
저장과 불러오기는 ConfigFile을 사용한다. 게임 시작 시 자동으로 불러오고, 재화나 장비가 변경될 때마다 자동 저장한다. 재화 변경 시 data_changed 시그널을 emit해서 UI가 자동으로 갱신되도록 한다.

Task 3. 장비 데이터 구조 구현
Equipment 클래스를 Resource로 구현한다. 필드는 다음과 같다.
gdscriptvar id: String
var name: String
var grade: int        # 0=Normal 1=Uncommon 2=Rare 3=Epic 4=Legendary 5=Mythic
var slot: String      # "weapon" "armor" "accessory" "bag"
var base_atk: float
var level: int
var skill_id: String  # 고유 스킬 식별자 (MVP에서는 빈 값 허용)
공격력 계산 함수를 구현한다. 공식은 base_atk × (1.15 ^ level)이다. 등급별 base_atk 기본값은 Normal=10, Uncommon=25, Rare=60, Epic=150, Legendary=400, Mythic=1000으로 설정한다.

Task 4. 연못(소환) 시스템 구현
PondManager.gd를 구현한다. 소환 1회 비용은 dust 100이다. 연못 레벨별 등급 확률 테이블을 딕셔너리로 구현한다. 레벨 1은 Normal 90%, Uncommon 10%. 레벨 5는 Normal 40%, Uncommon 45%, Rare 10%, Epic 5%. 레벨 10은 Normal 10%, Uncommon 25%, Rare 40%, Epic 20%, Legendary 4.9%, Mythic 0.1%. 레벨 20은 Normal 1%, Uncommon 4%, Rare 15%, Epic 40%, Legendary 35%, Mythic 5%.
소환 결과로 Equipment 인스턴스를 반환한다. 환원(분해) 함수를 구현한다. 장비를 분해하면 등급에 따라 sacred_water가 지급된다. Normal=1, Uncommon=3, Rare=8, Epic=20, Legendary=60, Mythic=200. 연못 레벨업 비용은 sacred_water로 충당하며, 레벨업에 필요한 sacred_water는 current_level × 10으로 계산한다.

Task 5. 전투 시스템 구현
5-1. Player 씬
CharacterBody2D 기반으로 구현한다. 자동으로 오른쪽으로 이동하며 (이동 속도 기본값 150), 사거리 내에 적이 있으면 멈추고 자동 공격한다. 공격력은 DataManager에서 현재 장착 장비의 ATK 합산값을 가져온다. 공격 속도는 초당 1회가 기본값이다. 체력이 0이 되면 BattleManager에 player_died 시그널을 emit한다.
5-2. Enemy 씬
CharacterBody2D 기반으로 구현한다. 등급은 normal, elite, boss 세 가지다. 체력과 공격력은 현재 챕터와 스테이지 번호를 기반으로 계산한다. 공식은 base_hp × (1.2 ^ stage_number)다. 처치 시 dust를 드롭한다. 드롭량은 등급에 따라 normal=5, elite=15, boss=50이다.
5-3. BattleManager
웨이브 진행을 관리한다. 스테이지 1개는 웨이브 5개로 구성된다. 웨이브 1~4는 normal 적 3~5마리, elite 적 1마리를 스폰한다. 웨이브 5(보스 웨이브)는 boss 1마리만 스폰한다. 웨이브 내 모든 적 처치 시 다음 웨이브로 자동 진행한다. 웨이브 5 클리어 시 stage_cleared 시그널을 emit하고 보상을 지급한다. player_died 신호 수신 시 현재 스테이지 처음(웨이브 1)으로 자동 리셋한다. 게임오버 없이 무한 재시도한다.

Task 6. 메인 홈 화면 구현
MainScene.tscn을 구현한다. 구성 요소는 다음과 같다.
상단 바: dust, sacred_water, seal 수치를 아이콘과 함께 표시한다. DataManager의 data_changed 시그널을 구독해서 자동 갱신한다.
중앙 연못 영역: 소환 버튼을 배치한다. 버튼 클릭 시 PondManager.summon()을 호출한다. 소환 결과로 장비 비교 팝업을 띄운다.
장비 비교 팝업: 새 장비와 현재 장착 장비의 ATK를 비교한다. ATK가 높으면 ▲ 초록색, 낮으면 ▼ 빨간색으로 표시한다. 버튼은 [장착], [환원], [닫기] 세 가지다.
하단 메뉴 바: [정령], [연못], [가방], [모험], [성좌] 5개 탭으로 구성한다. MVP에서 [모험] 탭만 실제 전투 씬으로 연결하고, 나머지는 빈 화면으로 둔다.

Task 7. 오프라인 보상 구현
DataManager에 last_exit_time을 저장한다. 게임 시작 시 현재 시간과 last_exit_time의 차이를 계산한다. 최대 인정 시간은 8시간(28800초)으로 제한한다. 오프라인 보상량은 초당 dust 1개를 기준으로 계산한다. 게임 시작 시 오프라인 보상 팝업을 띄워 "X시간 동안 dust Y개를 모았습니다"를 표시하고 [수령] 버튼으로 지급한다.

Task 8. 자동 환원 설정 구현
설정 화면에 자동 환원 등급 기준을 추가한다. 유저가 "Normal 이하 자동 환원" 같은 기준을 설정할 수 있다. 소환 결과가 기준 이하 등급이면 장비 비교 팝업 없이 자동으로 환원 처리하고 상단에 작은 토스트 메시지로 "Normal 장비가 환원되었습니다. +3 성수"를 표시한다.

Task 9. 챕터/스테이지 진행 구현
챕터는 2개, 챕터당 스테이지 10개로 총 20스테이지를 구현한다. 챕터 1 배경은 오염된 숲, 챕터 2 배경은 무너진 항구로 설정한다. 스테이지 클리어 시 다음 스테이지가 잠금 해제된다. 모험 탭에서 잠금 해제된 스테이지 목록을 표시하고, 클리어한 스테이지는 별 표시로 구분한다. 현재 최고 스테이지에서 자동 전투가 반복 진행되도록 한다.

Task 10. 빌드 및 최종 점검
Android 빌드 설정을 완료한다. 다음 항목을 점검한다. DataManager 저장/불러오기 정상 작동 여부. 오프라인 보상 시간 계산 정확도. 소환 확률이 테이블과 일치하는지 1000회 시뮬레이션 후 확인. 전투 중 앱을 백그라운드로 보낼 때 last_exit_time이 정상 저장되는지 확인. 저사양 기기 기준(RAM 2GB) 프레임 드롭 여부 확인.
